# Template: Narrative Longread Synthesis (любой домен, запрос «понять поле»)

> Added 2026-07-18 after user feedback on 2026_07_emergence_design: the health-style
> 12-section synthesis produced «сухо, непонятно, выдуманная терминология, полной картины нет»
> for an ideas/field-map topic. THIS template replaces it for non-health domains.
> Reference implementation: `01_library/research/systems_thinking/automated_reviews/2026_07_emergence_design/synthesis_ru.md`.

## When to use

- **Триггер — тип запроса, не домен:** «понять поле / карта темы / изучить X» (мыслители,
  работы, кейсы, история идеи) → лонгрид. **Любой домен**: company, science, macro,
  creative, marketing, mixed.
- **Decision-исследования** («принимать ли X», «входить ли в Y», go/no-go) — в ЛЮБОМ домене
  остаются на структурном 12-секционном шаблоне v3.10 (там нужны грейды, decision tree, мониторинг).
- **Health** — всегда 12-секционный (безопасность, confidence-дисциплина, cross-protocol),
  даже для обзорных тем; повествовательность можно добавлять внутри секций, не вместо них.

## Hard rules (нарушение = провал синтеза)

1. **ЗАПРЕТ выдуманной терминологии.** Внутренние термины рисёрча (seams/«швы», cells/«клетки»,
   keystone/«замковый камень», container, loop-gain, «локусы») НЕ переносятся в user-facing текст
   как термины. Явление описывается обычными словами («правила стыков между командами»,
   «основатель-арбитр», «кто владеет системой снаружи», «цикл делает обороты без внешнего толчка»).
   Термин допустим только если он общепринят вне этого рисёрча — и тогда вводится по-русски через пример.
2. **Истории > тезисы.** Каждый ключевой вывод несёт минимум одну рассказанную историю с
   конкретикой (NUMMI: >85% тех же рабочих, 3 месяца; VISA: 37 лет → IPO). Кейсы из стримов
   доезжают до синтеза КАК ИСТОРИИ, не как «свидетельства к гипотезе».
3. **Личный перенос — только если явно просили.** Если запрос был «карта поля» — личное
   приложение ≤ 1 страницы В КОНЦЕ (или отсутствует), не стержень документа. Непрошенный
   transfer-layer как основа синтеза = ошибка scope.
4. **Один автор, один проход.** Финальный русский текст пишет ОРКЕСТРАТОР сам (не сборка
   чанков субагентов) — единый голос, проверенный рендер. Субагенты дают материал, не финальный текст.
5. **Числа — только верифицированные** (после fact-check/evidence-rescue), с лёгкими пометками
   уверенности («уверенность высокая», «слушать, деля на два») вместо плотного (conf 0.XX) на каждой фразе.
   Полная карта утверждений с card-id живёт в технической версии/консенсусе — ссылка в шапке.
6. **Старую версию не удалять** — переименовать в `synthesis_[lang]_v1_technical.md`.

## Structure (секции — ориентир, не жёсткий каркас)

```
## 1. TL;DR                    — 5-10 строк человеческим языком (для нотификаций)
## 2. Сама идея                — вопрос темы, откуда он у юзера
## 3. История мысли            — кто до этого додумался: линии как рассказ (кто, что открыл, чем неожиданно)
## 4. Самая строгая ветка      — где тема стала наукой (теоремы, работающие внедрения, провалы теории)
## 5. Кейсы-воплощения         — истории с конкретными правилами и цифрами, включая финалы историй
## 6. Провалы и мифы           — развенчания, признания инсайдеров, что у провалов общего
## 7. Что сходится             — 4-6 выводов простыми словами, каждый через пример
## 8. Насколько верить         — слои доказательности по-простому + честность о самом рисёрче
## 9. Личное приложение        — ≤1 стр, ТОЛЬКО если просили (иначе пропустить)
## 10. Куда смотреть дальше    — фронтир по-простому + порядок чтения книг
## So What Test                — 3 пункта, без жаргона (стандарт)
## Read-back Test              — стандарт
## Связанные файлы
```
